# Seminar Management System - 模块功能逻辑

## 1. 用户认证模块

### 登录流程
```
1. 显示登录对话框 (Modal Dialog)
2. 用户选择角色类型: Student / Evaluator / Coordinator
3. 输入用户名和密码
4. 验证凭据 → 成功: 切换角色视图 / 失败: 显示错误提示
5. 更新侧边栏菜单和权限
```

### 登出流程
```
1. 确认登出对话框
2. 清除当前会话
3. 重置为 Guest 模式
4. 跳转到日程查看页面
```

## 2. 研讨会日程模块 (所有角色)

- 日历视图展示研讨会场次

## 3. 学生注册模块 (Student)

### 注册表单字段
```java
class RegistrationForm {
    String researchTitle;      // 研究标题 (必填, 最大200字符)
    String abstractText;       // 摘要 (必填, 最大1000字符)
    String supervisorName;     // 导师姓名 (必填)
    PresentationType type;     // 演示类型: ORAL / POSTER
    Long selectedSessionId;    // 选择的场次 (必填)
}
```

### 注册流程
```
1. 填写研究信息
2. 选择演示类型 (Oral/Poster)
3. 选择可用场次 (仅显示 OPEN 状态)
4. 提交注册
5. 等待审批 (如需要) 或 直接确认
```

### 注册状态
```java
enum RegistrationStatus {
    PENDING,    // 待审批 - 黄色
    APPROVED,   // 已批准 - 绿色
    REJECTED,   // 已拒绝 - 红色
    CANCELLED   // 已取消 - 灰色
}
```

## 6. 场次管理模块 (Coordinator)

### 场次表单

```java
class SessionForm {
```

## 8. 奖项管理模块 (Coordinator)

### 奖项类型
```java
enum AwardType {
    BEST_ORAL,          // 最佳口头报告
    BEST_POSTER,        // 最佳海报展示
    PEOPLES_CHOICE      // 最受欢迎奖
}
```

### 计算逻辑
```
最佳口头报告: Oral类型中平均分最高者
最佳海报展示: Poster类型中平均分最高者
最受欢迎奖: 投票数最多者 (需要投票功能)
```
